# NMC 天气

面向 Android 12 及以上的轻量天气 App。主天气来自中央气象局；广州可选南沙区气象局区级实况，和风天气仅用于可选的分钟级降雨预测。

## 功能

- 当前天气：温度、体感、湿度、风、气压和空气质量。
- 24 小时实况：整点温度、小时降水量和湿度。
- 未来 5 天：最高/最低温曲线、天气、风向和风力。
- 未来 1 小时降雨曲线：每 5 分钟一个点，并估算未来 30 分钟降雨概率。
- 雷达：8 个区域拼图；31 个省级分类、268 个中央气象局单站。
- 透明 2×1 桌面组件：显示当前温度、天气与图标。

## 数据来源

### 中央气象局

以下内容只使用中央气象局：

- 当前天气
- 24 小时实况
- 未来 5 天
- 桌面组件
- 区域雷达和单站雷达

### 南沙区气象局（广州专用）

在广州的行政区列表中可选择“南沙区（南沙天气）”。App 会在前台加载南沙区气象局动态页面，优先使用南沙的更新时间、温度、湿度、小时雨量、风向风力和可识别的当日预报。体感、气压、AQI、24 小时实况及缺少的五日数据由番禺 NMC 补齐。

南沙页面为空、过期或加载失败时自动回退番禺，并在前台提示。后台更新不创建隐藏 WebView，直接使用番禺 NMC 同步桌面组件。

### 和风天气（可选）

和风天气只用于：

- GeoAPI：把中央气象局中选择的城市或区县转换为经纬度。
- 分钟降水：生成未来 1 小时降雨曲线。

不再使用和风天气的当前天气、24 小时预报、7 日预报或行政区列表。凭据只需启用 **GeoAPI** 和 **分钟降水** 权限。

不填写 API Key 和 Host 时，主天气和雷达仍可正常使用；降雨区域显示“点击3次启动降雨预测”。

## 更新规则

- 冷启动：立即更新一次。
- 下拉刷新：立即更新天气、降雨预测和雷达。
- App 持续在前台：约每 30 分钟更新。
- 后台：约每 30 分钟只更新中央气象局当前天气并同步桌面组件，不加载雷达网页。
- 连续两次后台更新失败后停止后台更新，直到用户主动下拉刷新。

## 设置

1. 选择中央气象局省份与城市。
2. 可选距离最近的中央气象局行政区/气象站；广州另有“南沙区（南沙天气）”。
3. 如需降雨预测，填写和风 API Key 与 API Host；两项必须同时填写。
4. 选择区域雷达，或按省份选择具体单站雷达。

所有下拉框最多同时显示 5 行，更多项目可滚动查看。

## 构建

- Android Studio
- Android 12+（minSdk 31）
- compileSdk / targetSdk 34
- Java 17

打开项目后等待 Gradle 同步，然后选择：

`Build → Build APK(s)`

APK 通常位于：

`app/build/outputs/apk/debug/app-debug.apk`

## 网络与存储

- 支持 WiFi 和移动数据。
- 仅对 `tqyb.com.cn` 开放 HTTP 明文访问，其他域名保持禁止。
- 不保存天气历史数据库。
- 设置和桌面组件最近一次天气保存在 SharedPreferences。
- 雷达网页缓存由 Android WebView 管理。
- 和风 API Key 与 Host 只保存在本机应用配置中，不应提交到 GitHub 源代码。
