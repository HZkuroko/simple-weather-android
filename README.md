# NMC 天气 (安卓 12+)

一个极简的天气 App，数据来自中国气象网 nmc.cn。

## 功能
- 首次打开弹窗选择天气城市与雷达；雷达可选区域，或按省份选择具体单站。
- 顶部：当前气温、天气、体感、湿度、风、气压、空气。
- 中部：24 小时实况、未来 5 天预报。
- 底部：只显示设置中保存的一个区域雷达或单站雷达；点顶部城市名可重新进入设置。
- **更新时机：后台约每 30 分钟更新一次；每次冷启动和下拉刷新也会立即更新，并同步桌面组件。**
- 支持 WiFi 与移动数据（仅声明 INTERNET 权限）。

## 技术
- Kotlin，minSdk 31 (安卓 12)，targetSdk 34。
- 无重型依赖：网络用 HttpURLConnection，JSON 用内置 org.json，异步用协程。
- 城市内部 code 由接口自动获取（不需手写拼音）。

## 接口说明
- 省份：`https://www.nmc.cn/rest/province/all`
- 城市：`https://www.nmc.cn/rest/province/{省code}`（如 AGD）
- 天气：`https://www.nmc.cn/rest/weather?stationid={城市code}`
- 雷达页：`https://www.nmc.cn/publish/radar/{地区}.html`

## 如何构建 APK（两种方式）

### A. Android Studio（推荐，最简单）
1. 用 Android Studio（Hedgehog / Koala 或更新）选择 **Open**，打开本文件夹 `NMCWeather`。
2. 等 Gradle 同步完成（首次会自动下载依赖，需联网）。
3. 菜单 **Build > Build Bundle(s) / APK(s) > Build APK(s)**。
4. 完成后点“locate”，得到 `app/build/outputs/apk/debug/app-debug.apk`，传到手机安装即可。

### B. 命令行
本工程未包含 gradle-wrapper.jar。若本机已装 Gradle 8.7+：
```bash
cd NMCWeather
gradle wrapper            # 生成 ./gradlew
./gradlew assembleDebug   # 需已装 Android SDK 并配置 ANDROID_HOME / local.properties
```
产物同样在 `app/build/outputs/apk/debug/`。

## 短临降水（和风天气 QWeather）

nmc.cn 本身不提供“未来分钟级降雨概率/降雨量曲线”，所以这个卡片用**和风天气**的分钟级降水接口实现。

### 注册并拿到 Key + API Host
1. 打开 https://dev.qweather.com 注册账号（免费）。
2. 进入控制台 → **项目管理** → 创建项目 → 创建**凭据**，凭据类型选 **API KEY**，复制这串 Key。
3. 在控制台“设置”里能看到你账号专属的 **API Host**（形如 `abcd1234.qweatherapi.com`）。
4. 打开 App，点顶部城市名，在弹窗**最底部**把 Key 和 API Host 粘进去，保存即可。

### 显示内容
- **未来 30 分钟**：根据前 6 个 5 分钟点判断是否有雨、强度，并给出一个**估算概率**（标注“估算”，非官方真实概率）。
- 官方 `summary` 文字提示（如“未来两小时不会下雨”），这是最权威的一句话结论。
- **未来 1 小时降水曲线**：每 5 分钟一个点（共 12 点），自绘轻量曲线，单位毫米。

### 准不准？
- 分钟级降水是**实况外推**，对“接下来 1~2 小时会不会下、大概什么时候下”比较准；时间越往后越不确定。
- 和风的分钟级预报覆盖中国大陆，逐 5 分钟更新，通常比 nmc 的“逐日/逐小时”预报更适合判断马上要不要带伞。
- App 里的百分比是我用降水量点位**估算**出来的，仅供参考；官方那句 `summary` 更可信。
- 免费“开发版”可按当前套餐权限使用相关接口；主天气可选 nmc 或和风天气，雷达始终来自 nmc。分钟降水失败时只在该卡片显示错误。

## 天气来源、行政区与单站雷达

- 初始设置可选择 **中央气象局** 或 **和风天气** 作为主天气来源。
- 选择和风天气时，只有 API Key 与 API Host 都不为空才会生效；任一留空会自动回退到中央气象局。
- 中央气象局与和风天气使用互相独立的行政区列表，切换来源不会混用站点：
  - 中央气象局列表来自其可用气象站，请选择距离最近的站点。
  - 和风列表通过 GeoAPI 按当前城市加载；也可保持“不选择”，使用城市级天气。
- 和风作为主天气来源时，会读取实况、24 小时与 7 天接口，并在 App 中显示前 5 天。
- 和风凭据的 API 权限需要启用：**GeoAPI、天气预报、分钟降水**。
- 雷达下拉保留 8 个区域拼图，并加入乌鲁木齐、深圳、北京大兴、上海南汇、郑州、青岛、太原等多个 nmc 单站雷达。
- 所有下拉弹窗最多同时显示约 5 行，更多项目在弹窗内滚动，避免超出屏幕。


## 更新频率与桌面组件

- App 在前台持续打开时，每 30 分钟刷新完整天气页面。
- App 关闭或处于后台时，由 Android WorkManager 约每 30 分钟获取当前天气并同步桌面组件。
- 每次冷启动和下拉刷新都会立即更新完整天气，并立即同步透明 2×1 桌面组件。
- 后台任务仅在有网络时执行；受 Android 省电调度影响，半小时任务可能有几分钟延迟，不保证精确到秒。
- 桌面组件显示天气图标、当前温度和天气文字，点击可打开 App。

## 后台失败保护

- 只统计半小时后台任务的连续失败，前台刷新失败不计入。
- 第一次后台失败后等待下一个半小时周期，不进行额外快速重试。
- 连续第二次后台失败后取消后台更新任务，保留桌面组件中的最后一份成功数据。
- 冷启动不会解除暂停；只有用户主动下拉刷新时才会清零失败次数并重新注册半小时后台任务。
- 在达到两次之前，只要后台成功一次，连续失败次数就会清零。

## 雷达选择结构

- 雷达只在初始设置/城市设置页面选择，主页面不再显示切换下拉框。
- 第一级选择“区域雷达”或“单站雷达”。
- 区域雷达第二级选择全国、华北、东北、华东、华中、华南、西南、西北。
- 单站雷达先选择省份/直辖市，再选择具体雷达站。
- 单站目录按中央气象局当前官方移动版目录整理，共 31 个省级分类、268 个站点，包含官网仍使用旧路径的站点。
- 主页面雷达卡片只加载并显示设置中保存的一个区域或单站，标题会显示当前选择名称。

## 缺失值修复

- 中央气象局接口中的 `9999` 表示数据缺失，不是天气状态或实际数值。
- 日预报天气文字现在会统一清洗；白天天气缺失时自动使用夜间天气，两者都缺失时显示“暂无”。
- 当前天气、风向、风力、温度与未来 5 天显示增加第二层缺失值保护，不再直接显示 `9999`。
